API 接口列表

方法 接口路径 功能描述 认证要求
GET /deposit/records 获取保证金购买记录列表，支持分页查询和条件筛选 ✅ 必需
POST /deposit/records 创建新的保证金购买记录，包含用户信息和支付详情 ✅ 必需
GET /deposit/records/:id 获取指定ID的保证金购买记录详细信息 ✅ 必需
POST /deposit/records/:id/refund 申请保证金退款，需要提供退款原因和相关信息 ✅ 必需
GET /deposit/statistics 获取保证金统计信息，包含总额、数量、趋势等数据 ✅ 必需
GET /deposit/export 导出保证金记录表格，支持多种格式（CSV、Excel） ✅ 必需
POST /deposit/callback 处理第三方支付平台的回调通知，更新订单状态 ❌ 不需要

🔐 认证信息

认证方式： Bearer Token (JWT)

请求头格式： Authorization: Bearer <jwt_token>

Token 获取： 通过用户登录接口获得

Token 有效期： 7天

请求参数说明

接口 参数名 类型 必需 说明
/deposit/records (GET) page number 否 页码，默认为1
limit number 否 每页数量，默认为10，最大100
status string 否 状态筛选：pending, active, refunded, expired
startDate string 否 开始日期，格式：YYYY-MM-DD
endDate string 否 结束日期，格式：YYYY-MM-DD
/deposit/records (POST) contractId string 是 契约ID
amount number 是 保证金金额，必须大于0
paymentMethod string 是 支付方式：alipay, wechat, bank_transfer, credit_card
description string 否 备注说明，最大500字符
/deposit/records/:id/refund reason string 是 退款原因

响应状态码

状态码 状态描述 说明
200 OK 请求成功
201 Created 资源创建成功
400 Bad Request 请求参数错误
401 Unauthorized 未授权，需要登录
403 Forbidden 禁止访问，权限不足
404 Not Found 资源不存在
429 Too Many Requests 请求频率超限
500 Internal Server Error 服务器内部错误
